# فہرست مضامین

## تعارف

مصنوعی ذہانت (AI) اور روبوٹکس کا سنگم ایک نئے دور کا آغاز کر رہا ہے—فزیکل اے۔ آئی کا۔ یہ کتاب، "فزیکل اے آئی اور ہیومنائیڈ روبوٹکس - لوازمات"، ذہین طبعی نظاموں کو سمجھنے اور بنانے کے لیے ایک جامع گائیڈ کے طور پر کام کرتی ہے، جس میں خاص طور پر ہیومنائیڈ روبوٹس پر توجہ دی گئی ہے۔ جیسے جیسے اے آئی خالصتاً ڈیجیٹل دائروں سے حقیقی دنیا میں مجسم ایجنٹوں کی طرف بڑھ رہی ہے، نئے چیلنجز اور مواقع ابھر رہے ہیں۔ ہیومنائیڈ روبوٹس، اپنے انسان نما فارم فیکٹرز کے ساتھ، انسانوں کے لیے بنائے گئے ماحول میں تعامل کرنے اور کام کرنے کے لیے منفرد طور پر پوزیشن میں ہیں، جو انہیں اے آئی کے مستقبل کے لیے ایک مرکزی نقطہ بناتے ہیں۔

*   **فزیکل اے آئی کیا ہے؟** فزیکل اے آئی ایک ایسا نمونہ ہے جہاں ذہین ایجنٹوں کو جسمانی جسم عطا کیے جاتے ہیں، جو انہیں ٹھوس دنیا میں سمجھنے، استدلال کرنے اور عمل کرنے کے قابل بناتے ہیں۔ روایتی اے آئی کے برعکس، جو اکثر مصنوعی یا تجریدی ڈومینز میں کام کرتی ہے، فزیکل اے آئی طبیعیات، حقیقی وقت کے حسی ان پٹ، اور متحرک ماحول کی پیچیدگیوں سے نبرد آزما ہوتی ہے۔
*   **ہیومنائیڈ روبوٹکس کیوں؟** ہیومنائیڈ روبوٹس انسانی مراکز میں کاموں کے لیے بے مثال استعداد پیش کرتے ہیں۔ ان کی دو ٹانگوں والی حرکت، مہارت سے کام کرنے والے، اور انسان نما تعامل کی صلاحیتیں انہیں سیڑھیاں چڑھنے، دروازے کھولنے، انسانی اوزار استعمال کرنے، اور دیگر روبوٹک شکلوں کے مقابلے میں زیادہ قدرتی طور پر سماجی تعاملات میں مشغول ہونے کی اجازت دیتی ہیں۔
*   **ہدف کے سامعین:** یہ کتاب روبوٹکس کے شوقین، اے آئی پریکٹیشنرز، انجینئرز، محققین، اور طلباء کے لیے ڈیزائن کی گئی ہے جو فزیکل اے آئی اور ہیومنائیڈ روبوٹکس کے بین الضابطہ میدان میں دلچسپی رکھتے ہیں۔ پروگرامنگ کے تصورات اور لکیری الجبرا کی بنیادی تفہیم فائدہ مند ہوگی۔

## حصہ 1: فزیکل اے آئی کی بنیادیں

فزیکل اے آئی، اپنی اصل میں، مصنوعی ذہانت کی حسابی طاقت کو روبوٹکس کے جسمانی مجسمے کے ساتھ مربوط کرتی ہے۔ یہ علامتی رشتہ مشینوں کو حقیقی دنیا میں سمجھنے، استدلال کرنے اور عمل کرنے کے قابل بناتا ہے، جو خالصتاً ڈیجیٹل نقالی سے آگے بڑھتا ہے۔ مضبوط اور ذہین طبعی نظام تیار کرنے کے لیے روبوٹکس اور اے آئی دونوں کے بنیادی عناصر کو سمجھنا بہت ضروری ہے۔

### باب 1: روبوٹکس کے بنیادی اصول

روبوٹکس فزیکل اے آئی کی بنیاد بناتا ہے، جو طبعی ماحول کے ساتھ تعامل کے لیے میکانزم فراہم کرتا ہے۔ روبوٹک اصولوں پر مہارت—مکینیکل ڈیزائن سے لے کر کنٹرول تھیوری تک—ناگزیر ہے۔

#### کائیمیٹکس اور ڈائنامکس

کائیمیٹکس روبوٹس کی حرکت کو ان قوتوں پر غور کیے بغیر بیان کرتی ہے جو ان کا سبب بنتی ہیں، پوزیشن، رفتار اور سرعت پر توجہ مرکوز کرتی ہیں۔ اس کے برعکس، ڈائنامکس ان حرکات کو روبوٹ پر کام کرنے والی قوتوں اور ٹارک سے جوڑتا ہے۔

**فارورڈ کائیمیٹکس مثال (2-DOF پلانر آرم):**

جوائنٹ اینگلز کو دیکھتے ہوئے، اینڈ ایفیکٹر کی پوزیشن کا تعین کریں۔

```python
import numpy as np

def forward_kinematics_2dof(l1, l2, theta1, theta2):
    """
    2-DOF پلانر بازو کے لیے اینڈ ایفیکٹر پوزیشن کا حساب کرتا ہے۔

    Args:
        l1 (float): پہلے لنک کی لمبائی۔
        l2 (float): دوسرے لنک کی لمبائی۔
        theta1 (float): ریڈین میں پہلے جوائنٹ کا زاویہ۔
        theta2 (float): ریڈین میں دوسرے جوائنٹ کا زاویہ۔

    Returns:
        tuple: اینڈ ایفیکٹر کے (x, y) نقاط۔
    """
    x = l1 * np.cos(theta1) + l2 * np.cos(theta1 + theta2)
    y = l1 * np.sin(theta1) + l2 * np.sin(theta1 + theta2)
    return x, y

# مثال کا استعمال
l1_val = 1.0
l2_val = 0.8
theta1_val = np.pi / 4  # 45 ڈگری
theta2_val = np.pi / 6  # 30 ڈگری

end_effector_pos = forward_kinematics_2dof(l1_val, l2_val, theta1_val, theta2_val)
print(f"اینڈ ایفیکٹر پوزیشن: {end_effector_pos}")
```

#### سینسرز اور ایکچویٹرز

**سینسرز** روبوٹس کو اپنے ماحول کو سمجھنے کی اجازت دیتے ہیں۔ عام اقسام میں شامل ہیں:
*   **ویژن سینسرز:** کیمرے (RGB، ڈیپتھ، سٹیریو) آبجیکٹ کا پتہ لگانے، شناخت کرنے، اور 3D میپنگ کے لیے۔
*   **پروپریوسپٹیو سینسرز:** انکوڈرز، IMUs (انرشیل میژرمنٹ یونٹس) روبوٹ کی اپنی حالت کے لیے (جوائنٹ اینگلز، اوریئنٹیشن، ایکسلریشن)۔
*   **پروکسیمٹی سینسرز:** لائیڈار، الٹراسونک، انفراریڈ فاصلے کی پیمائش اور رکاوٹوں سے بچنے کے لیے۔

**ایکچویٹرز** روبوٹس کو عمل کرنے کے قابل بناتے ہیں۔ وہ برقی توانائی کو جسمانی حرکت میں تبدیل کرتے ہیں۔
*   **موٹرز:** ڈی سی موٹرز، سروو موٹرز، سٹیپر موٹرز۔
*   **ہائیڈرولک/نیومیٹک سسٹمز:** زیادہ قوت والی ایپلی کیشنز کے لیے۔

#### روبوٹ آپریٹنگ سسٹم (ROS) کے بنیادی اصول

ROS روبوٹ سافٹ ویئر لکھنے کے لیے ایک لچکدار فریم ورک فراہم کرتا ہے۔ یہ ٹولز، لائبریریوں، اور کنونشنز کا ایک مجموعہ ہے جو پیچیدہ اور مضبوط روبوٹ رویوں کو بنانے کے کام کو آسان بناتا ہے۔

**ROS 2 نوڈ مثال (Python):**

ROS 2 میں ایک سادہ پبلشر نوڈ جو "ہیلو ورلڈ" پیغامات بھیجتا ہے۔

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalPublisher(Node):
    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = f'ہیلو ورلڈ: {self.i}'
        self.publisher_.publish(msg)
        self.get_logger().info(f'پبلشنگ: "{msg.data}"')
        self.i += 1

def main(args=None):
    rclpy.init(args=args)
    minimal_publisher = MinimalPublisher()
    rclpy.spin(minimal_publisher)
    minimal_publisher.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

### باب 2: روبوٹکس میں مصنوعی ذہانت

اے آئی روبوٹس کو ذہانت سے بااختیار بناتی ہے، جس سے وہ سیکھنے، اپنانے، اور فیصلے کرنے کے قابل ہوتے ہیں۔

#### کنٹرول کے لیے مشین لرننگ

واضح پروگرامنگ کے بجائے، ایم ایل بہترین کنٹرول پالیسیاں سیکھ سکتا ہے۔
*   **رینفورسمنٹ لرننگ (RL):** روبوٹس آزمائش اور غلطی کے ذریعے سیکھتے ہیں، انعامات کی بنیاد پر اعمال کو بہتر بناتے ہیں۔
*   **امیٹیشن لرننگ:** روبوٹس انسانی مظاہروں کا مشاہدہ کرکے سیکھتے ہیں۔

#### پرسیپشن کے لیے کمپیوٹر ویژن

کمپیوٹر ویژن روبوٹس کو اپنے اردگرد کو "دیکھنے" اور اس کی تشریح کرنے کی اجازت دیتا ہے۔
*   **آبجیکٹ ڈیٹیکشن:** اشیاء کی شناخت (جیسے YOLO، R-CNN)۔
*   **سیمنٹک سیگمنٹیشن:** تصویری علاقوں کی پکسل سطح پر درجہ بندی۔
*   **3D ری کنسٹرکشن:** 2D تصاویر سے 3D ماڈل بنانا (جیسے SLAM)۔

**اوپن سی وی مثال (ایج ڈیٹیکشن):**

```python
import cv2
import numpy as np

def detect_edges(image_path):
    """
    ایک تصویر پر کینی ایج ڈیٹیکشن انجام دیتا ہے۔

    Args:
        image_path (str): ان پٹ تصویر کا راستہ۔

    Returns:
        numpy.ndarray: پتہ لگائے گئے کناروں والی تصویر۔
    """
    img = cv2.imread(image_path, cv2.IMREAD_GRAYSCALE)
    if img is None:
        print(f"خرابی: {image_path} پر تصویر لوڈ نہیں ہو سکی")
        return None

    # شور کو کم کرنے اور کنارے کا پتہ لگانے میں مدد کے لیے گاوسی بلر کا اطلاق کریں

    # کینی ایج ڈیٹیکشن انجام دیں
    edges = cv2.Canny(blurred_img, threshold1=50, threshold2=150)
    return edges
```

## حصہ 2: ہیومنائیڈ روبوٹکس

ہیومنائیڈ روبوٹکس فزیکل اے آئی میں ایک سرحد کی نمائندگی کرتا ہے، جس کا مقصد ایسی مشینیں بنانا ہے جو انسانوں کے لیے بنائے گئے ماحول میں کام کر سکیں۔ یہ سیکشن ان منفرد چیلنجز اور حلوں پر غور کرتا ہے جو بائپیڈل روبوٹس کو ڈیزائن کرنے، کنٹرول کرنے اور پیچیدہ انسانی مراکز میں تعامل کرنے کے قابل بنانے میں شامل ہیں۔

### باب 3: ہیومنائیڈ روبوٹ اناٹومی

ہیومنائیڈ روبوٹس کا مکینیکل اور الیکٹریکل ڈیزائن انتہائی پیچیدہ ہے، جس میں انسانی شکل اور کام کی نقل کرنے کے لیے محتاط انجینئرنگ کی ضرورت ہوتی ہے۔

#### مکینیکل ڈیزائن

*   **جسمانی ساخت:** ہلکے وزن کے باوجود مضبوط مواد (جیسے ایلومینیم الائے، کاربن فائبر) تاکہ اعلی طاقت سے وزن کا تناسب حاصل کیا جا سکے۔ دیکھ بھال اور اپ گریڈ میں آسانی کے لیے ماڈیولر ڈیزائن۔
*   **جوائنٹس اور ڈگریز آف فریڈم (DoF):** ہیومنائیڈز میں عام طور پر 20-60+ DoF ہوتے ہیں، جو پیچیدہ حرکات کو ممکن بناتے ہیں۔ ہر جوائنٹ کے لیے موٹرز، گیئرز، اور سینسرز کا محتاط انتخاب ضروری ہے۔
*   **اینڈ ایفیکٹرز:** ہاتھ اور پاؤں جو پکڑنے، ہیرا پھیری، اور مستحکم حرکت کے لیے بنائے گئے ہیں۔ اکثر نازک تعامل کے لیے ٹیکٹائل سینسرز شامل ہوتے ہیں۔

#### پاور سسٹمز

پیچیدہ ہیومنائیڈ حرکات کو موثر اور محفوظ طریقے سے طاقت دینا ایک بڑا چیلنج ہے۔
*   **بیٹریاں:** پائیدار آپریشن کے لیے اعلی توانائی کی کثافت (Li-Po، Li-ion)۔
*   **پاور ڈسٹری بیوشن:** متعدد ایکچویٹرز میں موثر وولٹیج ریگولیشن اور کرنٹ مینجمنٹ۔
*   **تھرمل مینجمنٹ:** موٹرز اور الیکٹرانکس سے پیدا ہونے والی حرارت کو ختم کرنا زیادہ گرمی سے بچنے اور لمبی عمر کو یقینی بنانے کے لیے بہت ضروری ہے۔

#### کمیونیکیشن آرکیٹیکچرز

*   **اندرونی کمیونیکیشن:** سینسرز، ایکچویٹرز، اور آن بورڈ کمپیوٹرز کے درمیان حقیقی وقت کے ڈیٹا کے تبادلے کے لیے تیز رفتار بسیں (جیسے EtherCAT، CAN بس)۔
*   **بیرونی کمیونیکیشن:** ریموٹ کنٹرول، ڈیٹا لاگنگ، اور کلاؤڈ کنیکٹیویٹی کے لیے وائرلیس ٹیکنالوجیز (Wi-Fi، بلوٹوتھ، 5G)۔

### باب 4: حرکت اور توازن

دو ٹانگوں والی حرکت فطری طور پر غیر مستحکم ہے، جس سے توازن اور مضبوط چال پیدا کرنا ہیومنائیڈ روبوٹکس کا مرکز ہے۔

#### چلنے کی چالیں

*   **زیرو مومنٹ پوائنٹ (ZMP):** مستحکم دو ٹانگوں والی چال کے لیے ایک بنیادی تصور۔ ZMP زمین پر وہ نقطہ ہے جہاں روبوٹ اس کے گرد کونیی رفتار پیدا کیے بغیر خالص قوت لگا سکتا ہے۔ استحکام کے لیے ZMP کو سپورٹ پولی گون کے اندر رکھنا بہت ضروری ہے۔
*   **پیٹرن جنریٹرز:** الگورتھم جو چلنے کے لیے سائیکلک جوائنٹ ٹریجیکٹریز پیدا کرتے ہیں، جو اکثر حسی فیڈ بیک کی بنیاد پر حقیقی وقت میں ایڈجسٹ ہوتے ہیں۔
*   **ڈائنامک واکنگ:** روبوٹ کی حرکیات کا فائدہ اٹھا کر زیادہ توانائی سے بھرپور اور قدرتی نظر آنے والی چالیں حاصل کرنا۔

**ZMP کیلکولیشن کا تصور:**

ZMP کو روبوٹ کی زمینی رابطہ قوتوں سے تصوراتی طور پر سمجھا جا سکتا ہے۔

```python
# ZMP کا تصوراتی حساب (مثال کے لیے آسان کیا گیا)
# حقیقت میں، اس میں پیچیدہ حرکیات اور فورس ٹارک سینسر ریڈنگز شامل ہیں۔

def calculate_zmp_simplified(cop_x, cop_y, fz_total, Mx, My, L):
    """
    ZMP کا آسان تصوراتی حساب۔
    اصل ZMP حساب میں روبوٹ کی جڑتا، سرعت، اور رابطہ قوتیں شامل ہیں۔

    Args:
        cop_x (float): سینٹر آف پریشر X-کوارڈینیٹ۔
        cop_y (float): سینٹر آف پریشر Y-کوارڈینیٹ۔
        fz_total (float): کل عمودی زمینی رد عمل قوت۔
        Mx (float): زمینی رد عمل قوتوں سے X-محور کے گرد مومنٹ۔
        My (float): زمینی رد عمل قوتوں سے Y-محور کے گرد مومنٹ۔
        L (float): حوالہ فاصلہ (جیسے روبوٹ کی اونچائی یا پاؤں کی لمبائی)۔

    Returns:
        tuple: (zmp_x, zmp_y) نقاط۔
    """
    if fz_total == 0:
        return float('inf'), float('inf') # کوئی زمینی رابطہ نہیں

    # بہت آسان تشریح؛ اصل مساواتیں زیادہ پیچیدہ ہیں۔
    zmp_x = cop_x - My / fz_total
    zmp_y = cop_y + Mx / fz_total
    return zmp_x, zmp_y

# مثال - عام طور پر فورس پلیٹ ڈیٹا یا فٹ سینسرز سے حاصل کیا جاتا ہے
cop_x_val, cop_y_val = 0.05, 0.02 # پاؤں کی اصل سے سینٹر آف پریشر
fz_val = 500.0 # نیوٹن میں کل عمودی قوت
Mx_val, My_val = 10.0, -5.0 # Nm میں مومنٹس
L_val = 0.1 # حوالہ لمبائی

zmp_result = calculate_zmp_simplified(cop_x_val, cop_y_val, fz_val, Mx_val, My_val, L_val)
print(f"آسان ZMP: {zmp_result}")
```

#### دو ٹانگوں کا استحکام

*   **توازن کنٹرول:** الگورتھم جو خلل کی موجودگی میں توازن برقرار रखने के लिए جوائنٹ ٹارک یا پاؤں کی جگہ کو ایڈجسٹ کرتے ہیں۔
*   **کیپچر پوائنٹ (CP):** ZMP سے زیادہ متحرک استحکام کا معیار، جو اس نقطہ کی نمائندگی کرتا ہے جہاں روبوٹ کو مکمل طور پر رکنے کے لیے قدم رکھنا چاہیے۔
*   **گرنے سے بحالی:** ناگزیر گرنے کے دوران نقصان کو کم کرنے کی حکمت عملی۔

#### جدید کنٹرول تکنیک

*   **ہول باڈی کنٹرول:** ایک ساتھ متعدد کاموں کو حاصل کرنے کے لیے تمام روبوٹ جوائنٹس کو مربوط کرنا (جیسے کسی چیز کو ہیرا پھیری کرتے ہوئے چلنا)۔
*   **ماڈل پریڈکٹیو کنٹرول (MPC):** روبوٹ اور اس کے ماحول کے ماڈل کا استعمال کرتے ہوئے مستقبل کی حالتوں کی پیش گوئی کرنا اور ایک پیچھے ہٹنے والے افق پر کنٹرول ان پٹس کو بہتر بنانا۔
*   **حرکت کے لیے رینفورسمنٹ لرننگ:** روبوٹس کو مصنوعی یا حقیقی دنیا کے تعاملات کے ذریعے چلنے اور توازن برقرار رکھنے کی تربیت دینا، جو اکثر انتہائی مضبوط اور چست چالوں کا باعث بنتا ہے۔

```python
        gesture_model: اشاروں کی شناخت کے لیے پہلے سے تربیت یافتہ مشین لرننگ ماڈل۔
```
